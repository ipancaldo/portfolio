About the project

This was my first and most complex project up to that point, which I completed using only C++.
It was my final project for the "OOP" subject in Computer Engineering, and I received a final score of 10/10.
It was later used in a small family business.

This is a CRUD application designed to list products scanned by a barcode scanner.

As a CRUD system, you can create, read, update, and delete the product list.

Additionally, you can manage your client list, place orders, and search for specific products.

When you start creating the product list, the first scanned (or manually entered) item will be assigned a ticket number, which is displayed above the list box on the left-hand side of the screen.

Once you have finished shopping, you can either save the list (by clicking 'Guardar') or print it (by clicking 'Imprimir').

The 'Guardar' button saves the list to the ticket history, while the 'Imprimir' button performs the same action and additionally creates a .txt file containing all the details in the \debug.w32 folder.

How I done it

At the beginning, I started brainstorming ideas for a new project.

Given my background in a merchant family, I thought that developing a sales system would be a good idea.

I began by writing down my ideas for the sales system in a Word file.

At the same time, I created an OpenOffice Calc (similar to Microsoft Excel but open source) to keep track of my progress. For each entry, I included the date, the amount of time spent, and a brief summary of what I had accomplished. This proved to be extremely useful.

Initially, I focused on coding the Product classes and then proceeded to implement the functionality to Add, Delete, and Save products. I continued to add more features to the system as I progressed."

Problems I had

As mentioned earlier, this was my first major project, and it presented several challenges that I will attempt to summarize and explain how I overcame them.

Graphical interface:

I created the graphical interface using the wxWidgets library, which was relatively easy to use.

Debugging:

I encountered several bugs throughout the development process, and this was my first experience using Stack Overflow. I spent a lot of time researching solutions to the issues I encountered, and I also consulted other forums, Google, and YouTube.

Data storage:

At the time, I did not know how to store data in SQL, so I had to improvise using binary and Notepad files. This proved to be a valuable learning experience for me.
